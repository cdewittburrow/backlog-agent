# First *Read*

*Twice a day. That's it.*

A personal daily news briefing app. Two editions per day — morning and evening — each generated once and locked until the next window opens. Read it, close it, go live your life.

---

## The problem

Most news products are optimized for engagement — more time in app, more return visits, more clicks. This one is optimized for the opposite.

The mental model is a physical newspaper, not an app. A newspaper doesn't refresh. It doesn't ping you. It sits on the table and waits. You can pick it up at 7am or 11am, read three articles or thirty, come back at lunch. When you're done, it doesn't care. Tomorrow there's a new one.

Apple News and Google News are opaque about sourcing and algorithmically tuned for engagement, not comprehension. There's no simple tool that gives you a daily briefing from *sources you explicitly trust* — with an editor's orientation, direct links, and no engagement mechanics.

## Stack

| Layer | Choice |
|-------|--------|
| Frontend | Vanilla HTML + JS modules (no build step) |
| Hosting | Vercel (free tier — static file serving, no functions needed) |
| Wire service news | Perigon API — AP and Reuters only |
| All other news | RSS feeds (28 sources) |
| Feed pipeline | GitHub Actions — fetches feeds twice daily, commits to repo |
| Edition generation | Claude scheduled agent (`claude-sonnet-4-6`) — triages, sections, writes editor's note |
| Client state | localStorage (edition cache + per-edition dismissed story state) |

## Architecture

Editions are pre-generated by a two-step pipeline and served as static JSON files. The browser fetches a static file — no serverless functions run at request time.

```
GitHub Action (5:50am MDT / 2:50pm MDT)
  → Fetch all 28 RSS feeds + Perigon (AP + Reuters)
  → Merge, deduplicate by URL, apply edition caps
  → Write data/feeds/{edition}-{date}.json
  → Commit + push to main

Claude scheduled agent (6:10am MDT / 3:10pm MDT)
  → Read data/feeds/{edition}-{date}.json from repo
  → Triage articles into named sections
  → Flag 3–4 stories worth reading first
  → Write 2–3 sentence editor's note
  → Write data/editions/{edition}-{date}.json
  → Commit + push to main (as github-actions[bot])

Vercel (auto-deploy on every push)
  → data/editions/ served as static files

Browser
  → GET /data/editions/{edition}-{date}.json
  → 404 → show "edition being prepared" state with previous edition
  → Cache in localStorage under edition key
  → Render: editor's note → named sections → story list
```

No API keys reach the browser. No serverless functions run on page load.

### Data flow

```
User opens app
  → Determine active edition key (e.g. "edition_morning_2026-04-18")
  → localStorage hit? → render immediately (zero network calls)
  → Cache miss:
      GET /data/editions/morning-2026-04-18.json
        → 200: cache + render
        → 404: edition not ready yet
            → Show previous edition + "Edition is being prepared…"
            → No retry loop — user refreshes manually when ready
```

### Edition model

| Window | Edition | File |
|--------|---------|------|
| ~6:10am → 3:00 PM MDT | Morning | `data/editions/morning-YYYY-MM-DD.json` |
| ~3:10pm → midnight MDT | Evening | `data/editions/evening-YYYY-MM-DD.json` |
| Midnight → 6:00 AM MDT | (overnight) | Shows last evening edition + note |

After midnight: the app shows the previous evening edition with a "New edition in the morning" note. No fetch.

### Article caps (EDITION_LIMITS)

```js
// scripts/fetch-feeds.js
morning: { national: 50, local: 10 }
evening: { national: 10, local: 40 }
```

Morning is nationally focused. Evening tilts local.

### Edition JSON schema

```json
{
  "key": "edition_morning_2026-04-18",
  "edition": "morning",
  "generated_at": "2026-04-18T12:14:00Z",
  "editor_note": "Senate Republicans...",
  "sections": [
    {
      "name": "Washington",
      "articles": [
        {
          "title": "...",
          "description": "...",
          "source_id": "AP News",
          "source_domain": "apnews.com",
          "link": "https://...",
          "pubDate": "2026-04-18T11:30:00Z",
          "tier": "national",
          "pill": "tc",
          "flagged": true
        }
      ]
    }
  ]
}
```

`flagged: true` on an article gives it the terracotta inset treatment. Set by the Claude agent.

### Story depth model

| Depth | What's visible |
|-------|---------------|
| 1 | Section header, source pill, timestamp, headline, lede. "Tap to expand" cue. Swipe right to dismiss. |
| 2 | + RSS body text verbatim, corroboration note, read link, collapse. |
| 3 | "Read on domain ↗" opens article at publisher in new tab. |

Stories are grouped into named sections (set by the Claude agent). Corroboration is computed client-side in `edition.js` using word-set overlap across headlines — no API call. Shown at depth 2 only.

### Swipe-to-dismiss

Swipe right on any story to check it off. The story collapses to a muted headline-only state with a terracotta ✓. Tap a dismissed story to restore it. Dismissed state persists in localStorage under `dismissed_{editionKey}` — coming back to the app later, your checkmarks are still there.

## Pipeline

### GitHub Action — fetch-feeds.yml

Runs at `50 11 * * *` (5:50am MDT) and `50 20 * * *` (2:50pm MDT).

1. Checks out the repo
2. Runs `scripts/fetch-feeds.js` with `PERIGON_API_KEY`
3. Commits `data/feeds/{edition}-{date}.json` and pushes to main

The feed file contains every article after deduplication and edition caps — ready for the Claude agent to read.

### Claude scheduled agent

Two triggers on Anthropic's infrastructure (no server to manage):

| Edition | Trigger ID | Cron (UTC) | Local time |
|---------|-----------|------------|------------|
| Morning | `trig_01CfzLuaVgiKy3XeSTyWXRAN` | `10 12 * * *` | 6:10am MDT |
| Evening | `trig_01L16KZF5zjj2zoaRX3t3A2c` | `10 21 * * *` | 3:10pm MDT |

The agent:
1. Reads `data/feeds/{edition}-{date}.json` from the repo
2. Groups articles into 4–6 named sections (Washington, Economy, Utah, etc.)
3. Flags 3–4 stories worth reading
4. Writes a 2–3 sentence editor's note (satellite radio style — names, not vibes)
5. Commits `data/editions/{edition}-{date}.json` using `github-actions[bot]@users.noreply.github.com` so Vercel accepts the deploy

Vercel auto-deploys on push. The static file is live within ~1 minute of the agent committing.

## AI prompt design

The Claude scheduled agent is the only AI in the pipeline. It reads raw feed JSON and writes the edition JSON.

### Agent prompt (condensed)

> You are the editorial agent for First Read. Read `data/feeds/{edition}-{date}.json`. Drop only: identical wire copy republished word-for-word, pure press releases, sports scores/recaps, celebrity gossip. Multiple outlets covering the same story is NOT a duplicate — it is corroboration; keep every outlet's version. Include ALL articles that pass the drop filter. Group them into 4–6 named sections ordered by newsworthiness. Mark 3–5 must-reads with `flagged: true`. Write a 2–3 sentence editor's note — declarative, name names, lead with the most significant story. Commit to `data/editions/{edition}-{date}.json`.

### Output schema

```json
{
  "key": "edition_morning_2026-04-18",
  "edition": "morning",
  "generated_at": "2026-04-18T12:14:00Z",
  "editor_note": "Senate Republicans failed to advance the reconciliation bill...",
  "sections": [
    {
      "name": "Washington",
      "articles": [ { "...": "...", "flagged": true } ]
    }
  ]
}
```

## API usage notes

- **Perigon:** 1 request per edition, AP + Reuters only. ~50 articles per wire service. Free tier: 150 requests/month. Two editions/day = ~60 calls/month.
- **RSS:** Direct HTTP fetches — no key, no rate limit, no cost. 28 feeds in parallel, 8s timeout per feed. Dead feeds silently drop.
- **Claude agent:** Runs on Anthropic's infrastructure as part of a Claude Code subscription. No per-call billing. Two runs/day.
- **Vercel:** Static file serving only — no function invocations at request time. Well within free tier.
- **GitHub Actions:** 2 runs/day × ~1 min each = ~60 min/month. Free tier is 2,000 min/month.

## Decision log

**Renamed to First Read**
Working title Pressbox dropped. "First Read" reflects the product's core mechanic — you read it first thing, and then you're done. Tagline: *Twice a day. That's it.*

**RSS-first, Perigon for wire services only**
AP News and Reuters have no usable RSS (AP killed theirs entirely; Reuters killed theirs June 2020). Every other source has a working RSS feed. By going RSS-first, Perigon's 100-article budget is spent entirely on AP + Reuters (~50 each) — a significant upgrade from the previous model where 100 articles split across 11 sources gave ~9 per source.

**Evening window at 3pm, not 5pm**
The evening edition is intended to feel like the evening news, not the end of the workday. 3pm is when people start winding down and checking in on the day. This also gives more runway before midnight.

**AI role narrowed to editor's note + sectioning**
Previous model: Gemini grouped stories into thematic sections and wrote a 2–4 sentence summary for each. New model: Claude writes 2–3 sentences orienting the whole edition, flags 3–4 stories worth reading first, and groups articles into named sections. The journalist's job (writing the story) stays at the publication. The editor's job (orienting the reader + organizing the paper) is the narrow AI task that's actually valuable here.

**Pipeline replaced serverless functions**
Original architecture: browser called `/api/news` (RSS + Perigon) and `/api/briefing` (Gemini) on every cache miss. This created Gemini rate-limit death spirals (20 RPD free tier), per-device quota exhaustion, and no way to guarantee edition consistency across devices. New architecture: GitHub Actions fetches feeds on a schedule, a Claude scheduled agent processes and writes the edition, Vercel serves a static JSON file. The edition is generated once and identical for every device. No serverless functions run at request time.

**Gemini replaced by Claude scheduled agent**
Gemini free-tier credits depleted, and the on-demand model had structural problems (see above). Claude scheduled agents run on Anthropic's infrastructure as part of a subscription — no per-call cost, no rate-limit death spirals, no quota management. The agent also does more: it sections the stories, not just writes the editor's note.

**Section headers restored (agent-grouped, not chronological)**
First iteration used flat chronological list — no sections, no AI grouping. This was clean but gave the reader no orientation. The Claude agent now groups articles into 4–6 named sections (Washington, Economy, Utah, etc.), which matches how a real paper is organized. The key difference from the original Gemini approach: the agent assigns each article to exactly one section and doesn't summarize individual stories.

**Per-article flagged boolean replaces flagged_indices array**
Original schema: `flagged_indices: [1, 4, 7]` — an array of positions in the flat article list. Fragile: any reordering broke the flags. New schema: `flagged: true` on the article object itself. Survives reordering, survives section restructuring, survives any future schema changes.

**Swipe-to-dismiss with persistent checkmark**
Swipe right on any story to check it off — collapses to a muted headline-only state with a terracotta ✓. Dismissed state persists in localStorage per edition key. Tap a dismissed story to restore it. This gives the "sit in my chair and work through the paper" use case — mark what you've read, come back later and pick up where you left off. The grandfather checkmark model: not deleting, just done.

**Feed audit — 7 broken URLs fixed, Google News proxy normalization**
Periodic audit (April 2026) found 7 of 28 feeds broken: Politico (403), HuffPost (404), KUER (ECONNREFUSED), Salt Lake Tribune (404), Deseret News (404), KSL News (malformed XML), KUTV (404). Salt Lake Tribune, Deseret News, and KSL had moved to new `arc/outboundfeeds` URLs. Politico, HuffPost, KUER, and KUTV either block bots or dropped native RSS entirely — all four switched to Google News RSS proxy. Google News proxy feeds get title normalization (strip trailing ` - Source Name`) and empty description (proxy provides no real article preview).

**Client-side corroboration detection**
Corroboration ("Also covered by AP and NPR") is computed in `edition.js` using word-set overlap across headlines — no API call, no server round-trip. Shown at depth 2 (expanded) only. Threshold: 3+ shared significant words (4+ chars, non-stopword). This is good enough for news headlines and adds zero latency.

**Three-depth story model**
Depth 1: headline + lede always visible. Depth 2: inline expand — shows RSS body verbatim, corroboration, read link. Depth 3: "Read on domain ↗" opens article at publisher. No API call at any depth after the initial edition fetch. The RSS body at depth 2 is shown verbatim — no AI rewriting, no summarization.

**Flagged stories**
Stories with `flagged: true` get a terracotta left border and subtle background tint — pulled slightly inset from the left edge so the border never touches the phone frame. Set by the Claude agent.

**Edition closing line**
Each edition ends with a plain closing sentence in italic serif:
- Morning: *"That's this morning. Evening edition after 3pm."*
- Evening: *"That's it for today. Morning edition tomorrow."*
This gives explicit permission to close the app.

**No WSJ**
WSJ's RSS feed returns headlines only — descriptions truncated due to paywall. Not useful for depth 2 body text. Dropped.

**No Washington Post**
Editorial direction concerns. Dropped.

**NYT kept despite headlines-only RSS**
The Times is useful for corroboration signal and headline scanning even without full body text. At depth 2, the thin-RSS fallback note explains honestly why body text isn't available.

**Tab bar removed, kept as commented reference**
The tab bar (Briefing / All Articles / Sources) has been removed from the visible UI. The HTML and JS remain commented in place so the old layout can be re-enabled for comparison.

**Sources sheet is read-only**
Sources are curated in `config/feeds.js` by the developer, not by the user at runtime. The sources sheet shows what sources contributed to the current edition (derived from the returned articles). Source editability is on the roadmap but needs more thought on UX before building.

**Agent corroboration prompt bug — fixed**
First real edition run (April 18, 2026 morning) produced only 9 articles from a feed of 50. Root cause: the agent prompt said "if 5 outlets covered the same press conference, keep the 2–3 best-sourced" — the agent read this as license to aggressively collapse any overlapping coverage, treating corroboration as redundancy. Fixed by inverting the instruction explicitly: multiple outlets covering the same story is corroboration, not a duplicate. Keep every outlet's version. Include ALL articles that survive the drop filter. The prompt now says this twice to make it unambiguous.

**Edition cache was always a miss — fixed**
`saveEdition` in `state.js` wrapped the edition object as `{ briefing: edition, ... }`, but the cache-hit check looked for `cached?.sections` (which lives at `cached.briefing.sections` in the wrapped format). Result: every page load was a fresh fetch, localStorage cache was write-only, and swipe-to-dismiss state didn't persist across sessions because the edition key was never reliably set. Fixed by storing the edition object directly and stamping `edition.key = key` in `app.js` before saving, so dismiss persistence always has the canonical key regardless of what the agent wrote.

**Swipe scroll conflict — fixed**
Swiping right to dismiss caused the page to scroll vertically at the same time. Fixed with `touch-action: pan-y` on `.story` — tells the browser to own vertical scrolling natively and pass horizontal touch events to JS. The two gestures no longer compete.

**Typography pass — larger sizes throughout**
First real-world use revealed text was too small for comfortable reading. Section headers: 9px → 11px, `ink-ghost` → `ink-muted`, weight bumped. Headlines: 17px → 18px. Lede: 13px → 14px. Expanded body: 12px → 13px.

## Roadmap

**v1.x — Refinement**
- PWA manifest — installable to iPhone home screen, offline shows last cached edition
- Skeleton loading states while edition generates
- Tune EDITION_LIMITS constants after living with the product
- Source editability UI — needs UX design before building; must be simple enough to not require documentation
- **Story thumbnails** — RSS audit (April 2026) confirmed ~half the feeds include image URLs via `media:thumbnail`, `media:content`, or `enclosure` fields. Sources with images: BBC, Guardian, Axios, The Hill, The Atlantic, NYT. Sources without: NPR, Politico, HuffPost, ProPublica, Mother Jones (and all Google News proxy feeds). Cost concerns are low: the GH Action would store only the URL string — no image bytes touch the pipeline, images load client-side. Claude context cost is trivial (~20 tokens per URL). The real design question is whether inconsistent image presence (roughly half the stories have art, half don't) enhances or clutters the typographic "morning paper" aesthetic. Proposed treatment: small right-side thumbnail only when the feed provides one; no placeholder for text-only stories — consistent with how print actually handles art. Do not build until the visual question is resolved.

**v2 — Threading & Archive**
- **Story threading** — evening edition notes when a morning story developed. Formally tabled — requires end-to-end eval of AI tool use before spec. Do not build prematurely.
- **Edition archive** — browse past editions. Requires Supabase or similar.
- **Coverage gap detection** — surface stories your sources missed vs. a broader baseline

**v3 — Bias & Trust Layer**
- Source bias labels — AllSides / Ad Fontes ratings per article
- Trust score per story — source count, bias spread, wire service pickup

**v4 — Reach**
- Daily digest email — Vercel cron generates and sends edition via Resend
- Multi-user — Supabase Auth, each user maintains own source list
