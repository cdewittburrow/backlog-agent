## Problem
When voting in Just Pick Already results in a tie, the app has no resolution mechanism. Users are left in a deadlock — the core promise of the app (just pick something) fails at the exact moment it's most needed.

## Goal
Automatically resolve tie votes with a randomized tiebreaker so every selection round produces a single winner without requiring manual user intervention.

## Users
Any participant in a Just Pick Already session where voting ends in a tie. Primary user is Chase, but includes any guests in a multi-participant session.

## Requirements

**Must have**
- When a vote round ends in a tie between two or more options, a tiebreaker is triggered automatically
- The tiebreaker selects a single winner via random selection (coin flip for 2-way tie, random draw for 3+ way tie)
- The result is clearly communicated to all participants

**Should have**
- The tiebreaker is animated — coin flip, spinner, or dice roll visual — so it feels deliberate and fun rather than silent/invisible
- The winning option is highlighted immediately after the tiebreaker resolves
- A short label like "It was a tie — fate decided!" contextualizes the outcome

**Nice to have**
- Option to re-roll the tiebreaker once (with confirmation, before the result is committed)
- Logging the tiebreaker outcome in session history so users can see it was random

## Out of scope
- Allowing users to configure the tiebreaker method
- Multi-round bracketing (e.g. run-off vote between tied options)
- Weighted randomness based on prior selections

## Open questions
- Is the tiebreaker triggered server-side or client-side? (Matters for multi-participant consistency — all users must see the same winner)
- Does the current data model track which options are tied, or just the winning option?
- Should guests be notified of the tiebreaker trigger, or does it happen silently for the host only?

## Decision log
- Random selection chosen over run-off vote — a run-off contradicts the "just pick already" ethos and adds friction
- Tiebreaker should be visible/animated to make the randomness feel intentional rather than like a silent app bug
