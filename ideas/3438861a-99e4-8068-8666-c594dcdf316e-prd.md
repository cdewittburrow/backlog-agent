## Problem

Homeowners accumulate a growing backlog of improvement and maintenance projects but have no tool that treats them as a true priority queue. Context (measurements, materials, shopping lists) lives in scattered notes, and deciding what to tackle next — given budget, season, and effort — requires mental overhead that most people avoid. The result is a passive list that never moves.

## Goal

Build a focused home project queue manager where AI helps the user decide what to do next based on cost, difficulty, and season — and where every project carries its own context (measurements, shopping list, action plan) so nothing falls through the cracks.

## Users

Primary: Chase (solo homeowner). Secondary: any homeowner who maintains a backlog of DIY improvement and maintenance projects. No contractor or marketplace use case in v1.

## Requirements

**Must have**
- Project card creation with title, description, and rough cost range ($ / $$ / $$$)
- User-defined difficulty rating per project (easy / medium / hard)
- User-defined ideal season per project (spring / summer / fall / winter / any)
- Freeform measurement notes field per project (text scratchpad, not structured fields)
- Simple per-project shopping list (item name + optional quantity, plain text)
- AI-generated prioritized project queue based on cost, difficulty, and season inputs
- AI-generated action plan per project (step-by-step task breakdown, user-requested on demand)
- Projects can be marked complete or archived
- Queue can be manually reordered by the user (overrides AI suggestion)

**Should have**
- Project status: not started / in progress / complete / on hold
- Notes/log field per project for ongoing progress notes
- Before/after photo capture per project
- Project categories (electrical, plumbing, landscaping, painting, general, etc.)
- Current season auto-detected from device date (no weather API needed in v1)

**Nice to have**
- Budget tracker: per-project actual spend vs. estimate
- Project dependency linking (project B unlocks after project A)
- Export project shopping list to clipboard or share sheet
- Reminders or nudges for projects with overdue seasons

## Out of scope

- Contractor marketplace integration or pro-hire workflows
- Structured measurement fields (area, volume, linear feet) with unit conversion — freeform notes only in v1
- Home Depot / Lowe's product search or price lookup
- Weather API or location-based seasonal intelligence — season is user-defined in v1
- Recurring maintenance reminders (separate category from project queue)
- Multi-user or household member assignment

## Open questions

1. **DIY scope assumption** — Does the AI plan assume the user does the work themselves, or should it detect when a project likely needs a contractor and flag it?
2. **AI action plan depth** — How detailed should the generated task breakdown be? Step-by-step with tool/material callouts, or just high-level phases?
3. **Queue ranking display** — Does AI prioritization produce a numbered ranked list, a scored card, or a "recommended next" single pick?
4. **Project dependency handling** — Can a dependent project be added to the queue before its dependency is done, or is it hidden until unlocked?
5. **Mobile-first or web?** — Stack and platform decision affects offline behavior for on-site use (e.g., at the hardware store without wifi).

## Decision log

- Freeform measurement notes chosen over structured fields for v1 — reduces build complexity and avoids forcing a schema before the use case is validated
- User-defined season chosen over weather API — eliminates an external dependency and is sufficient for the core prioritization use case
- No contractor integration in v1 — keeps the app focused on personal project management; contractor scope would change the product category
- AI prioritization based on three inputs (cost, difficulty, season) to keep the ranking legible and editable by the user
