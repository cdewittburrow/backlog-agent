## Problem
First Read displays RSS feed content inconsistently. Axios sends the full article body as their feed content, while most other sources only send the first paragraph. This means Axios stories show as walls of text in the lede view, while other sources show a truncated snippet that repeats what the headline already said. The result is an inconsistent reading experience across sources.

## Goal
Normalize the content extracted from RSS feeds so every story shows a consistent, useful lede — long enough to inform but short enough to scan. This is a prerequisite for several downstream improvements (image support, second-view expansion, etc.).

## Users
Chase, as the primary First Read user, reading news in a quick-scan mode. The goal is frictionless skimming — consistent snippets that let him decide read/skip without extra effort.

## Requirements

**Must have**
- Inventory all current RSS sources and document their feed schema (content field, summary field, description field, etc.)
- Define a normalized content schema: a single `lede` field capped at ~2–3 sentences or ~300 characters
- Extraction logic that picks the right source field per feed (or applies truncation where needed)
- Axios specifically must truncate to lede length, not show full article

**Should have**
- Per-source configuration so new feeds can declare their schema type
- Strip HTML tags from feed content before displaying

**Nice to have**
- Detect and deduplicate lede text that exactly matches the headline
- Flag sources with unusually sparse or rich content for manual review

## Out of scope
- Fetching full article content from the source URL (would require scraping)
- AI-generated summaries
- User-configurable lede length

## Open questions
- How many RSS sources are currently configured in First Read?
- Is content normalization done client-side or server-side (or at fetch time)?
- Does the current data model store raw feed content or already-processed content?

## Decision log
- Scope limited to normalization of existing feed data, not augmentation — keeps this as a data cleanup task rather than an AI feature
