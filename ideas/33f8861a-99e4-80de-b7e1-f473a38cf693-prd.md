## Problem
Just Pick Already currently requires the host to add all options themselves before a selection round begins. If the host is trying to get group input — for example, picking a restaurant with friends — everyone has to communicate their suggestions outside the app (text message, verbal, etc.) and the host manually enters them. This breaks the collaborative promise of the app and adds friction.

## Goal
Allow guests to submit their own suggestions to a Just Pick Already session before the final selection happens. The host retains control over when the session closes and selection begins. Guest submissions happen without requiring app installs or accounts if possible.

## Users
- **Host (Chase):** Creates the session, sets the category/context, shares the link, reviews submissions, closes intake, runs the final selection.
- **Guests:** Join via a shared link, submit one or more suggestions, see a confirmation that their suggestion was received. Low-friction experience — ideally no account required.

## Requirements

**Must have**
- Host can create a session and generate a shareable intake link
- Guests can access the intake form via that link (no app install required — web-based)
- Guests submit a name/label for their suggestion (e.g., a restaurant name or place)
- Host sees all submitted suggestions in the Just Pick Already session
- Host has explicit control over when intake closes and final selection begins
- Host can remove or edit any submitted suggestion before locking the round

**Should have**
- Guest can optionally add their name so the host knows who suggested what
- Basic validation: prevent empty submissions, duplicate names (case-insensitive)
- Intake link can be set to expire (e.g., after 24 hours or when host closes it)
- Host receives a notification or sees a live count when new suggestions come in

**Nice to have**
- Guest can see the other submissions already in the list (or keep them hidden until host closes intake — configurable)
- Guests can upvote / emoji-react to existing suggestions before intake closes
- Host can set a maximum number of suggestions per guest

## Out of scope
- Guest accounts or persistent guest identity across sessions
- Guests being able to modify or delete other guests' submissions
- Voting or ranking during intake (that is the selection phase)
- Real-time multiplayer selection flow in v1

## Open questions
- Does Just Pick Already currently have a backend, or is it fully local/client-side? The answer determines whether a shared link is feasible without a backend.
- What is the current session/options data model?
- Should the intake link be a web URL (requires web build or separate web form) or a deep link into the app?
- Is there a preferred sharing mechanism (SMS, clipboard, QR code)?

## Decision log
- Guest experience must be web-based and require no app install to maximize participation — requiring install would defeat the purpose
- Host retains full editorial control over the option list before selection begins
