## Overview

Recipes are not loading in the Recipes app — core read functionality is broken. This is a higher-severity bug than the import failure since it affects all stored recipes, not just the import path. This spec covers investigation approach, likely failure layers, and fix path.

## Architecture

The recipes loading path likely involves:
1. App/screen mounts, triggers data fetch
2. Fetch queries local DB or remote API/backend
3. Response is parsed and normalized into app state
4. State is passed to list/grid component for render
5. Individual recipe cards render from state

A blank or empty screen (vs. a crash) suggests the fetch is completing silently with no data rather than throwing. An error state or spinner stuck loading suggests the fetch is failing or hanging.

## Stack + dependencies

- Data layer: local SQLite / AsyncStorage / IndexedDB, or a remote backend (Supabase, Firebase, custom API)
- State management: React state, Zustand, Redux, or similar
- UI framework rendering the recipe list
- Auth layer — if recipes are user-scoped, an expired or missing session token would cause an empty or 401 response

## Data model

Recipes are likely stored with:
- `id`, `title`, `ingredients[]`, `steps[]`, `source_url`
- `user_id` — if multi-user or auth-scoped
- `created_at`, `updated_at`

If the query is filtering by `user_id` and the auth session is stale, the query returns 0 rows rather than erroring.

## Key implementation decisions

**Investigation order:**
1. Check auth state — if recipes are user-scoped, verify the session is valid and the user ID is being passed correctly to the query
2. Check network/API response — open network tab or add logging to confirm the fetch fires, what it returns (200 with empty array? 401? 500?), and whether the response shape matches what the UI expects
3. Check data layer — if using a local DB, confirm the table exists and has rows; a migration issue could have wiped or renamed the table
4. Check state management — confirm the fetched data is being written to state and that the component is subscribed to the right slice
5. Check render logic — confirm the list component renders when the array is non-empty; a conditional guard like `if (!recipes)` vs `if (!recipes.length)` can cause a blank render on empty-but-defined state

**Most likely culprits in order:**
1. Auth session expired → empty user-scoped query result
2. API schema change → response shape doesn't match what the UI destructures
3. State not being set after fetch (missing `setState` call or wrong action dispatched)
4. Database migration issue wiped or renamed the recipes table

**Fix:** Once root cause is identified, fix is likely 1–10 lines. Add a non-empty error boundary and loading/error state UI if not already present.

## Risks + unknowns

- If this is a production data loss (DB wipe via migration), recipes may be unrecoverable without a backup
- If auth is the issue, the fix is trivial but every user is affected until deployed
- "Not loading" could mean blank screen, infinite spinner, or error message — each points to a different layer

## Effort estimate

- Investigation: 30 minutes–2 hours
- Fix (auth/state/query issue): 1–3 hours
- Fix (schema mismatch requiring migration): half day
- Fix (data loss recovery): varies significantly

Total likely range: **2–4 hours** assuming no data loss.
