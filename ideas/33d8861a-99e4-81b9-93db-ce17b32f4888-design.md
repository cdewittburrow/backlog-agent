## Overview
We are evaluating and implementing image support for First Read story cards. This feature is gated on a feed audit — if fewer than ~60% of sources expose images natively, we defer rather than build a scraper fallback. Assuming the audit passes, the output is thumbnail images on story cards.

## User flows

**Normal feed load (with images):**
1. User opens First Read
2. Feed loads — story cards render with headline, lede, and thumbnail
3. Images load lazily as user scrolls
4. Cards without images render cleanly as text-only (no broken placeholder)

**Audit flow (one-time, developer-facing):**
1. Developer runs feed audit script against all configured sources
2. Script outputs: source name, image field found (or none), sample URL
3. Decision made: proceed or defer

## Information architecture

**Feed source config (updated):**
- Add `imageField` to per-source config: `media:content`, `enclosure`, `itunes:image`, or `null`
- Audit script populates this during initial discovery

**Story data model (updated):**
- Add optional `imageUrl: string | null` field to story
- Populated at parse time from the configured `imageField`
- `null` if source does not expose images or field is missing for this item

**Story card component (updated):**
- Conditionally renders thumbnail if `imageUrl` is present
- No-image state: existing layout unchanged (no placeholder)

## Interaction patterns

**Story card with image:**
- Thumbnail displayed at fixed size (e.g., 80×80px) on trailing edge of card, or as a banner above the headline — TBD based on existing card layout
- Tapping image navigates same as tapping the card (opens story)
- Images are lazy-loaded with a neutral background while loading

**Story card without image:**
- Renders exactly as today — no visual gap, no favicon fallback in v1

## Edge cases + empty states

- **Image URL returns 404 or fails to load:** hide image, render text-only card
- **Image is extremely wide or tall:** crop to thumbnail aspect ratio (do not stretch)
- **Source exposes image field but URL is malformed:** treat as null, skip
- **All stories in feed have no images:** feed renders text-only, no layout shift
