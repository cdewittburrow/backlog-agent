## Problem

There is no daily puzzle habit tool in the personal app portfolio, and existing options (NYT Games, Sudoku.com) carry ads, subscription prompts, and aesthetic choices that don't align with a personal daily ritual. The user wants something they can open each morning, complete in a few minutes, and feel good about — without the clutter of a commercial product.

## Goal

Build a personal, aesthetically clean daily Sudoku game as a portfolio piece and personal habit tool. Success means the user opens it daily and completes puzzles consistently. Distribution and user acquisition are explicitly not goals.

## Users

Primary: Chase (sole user). Secondary audience: anyone who sees it as a portfolio piece — potential collaborators or people curious about the stack. No onboarding, monetization, or multi-user session requirements.

## Requirements

**Must have**
- Daily puzzle: one new procedurally generated Sudoku per calendar day, consistent across sessions (seed-based so the same day always produces the same puzzle)
- Difficulty selection: at minimum Easy / Medium / Hard
- Completion state: clear success feedback when puzzle is solved
- Error indication: some signal when a placed number conflicts with rules (not necessarily real-time — could be on submit or toggle-able)
- Mobile-first layout: playable on phone one-handed

**Should have**
- Timer: optional display of elapsed time per puzzle
- Streak tracking: consecutive days completed, stored locally
- Notes/pencil mode: ability to mark candidate numbers in a cell
- Persistent progress: puzzle state saved so closing and reopening resumes where left off

**Nice to have**
- Light/dark mode
- Haptic feedback on correct/incorrect placement (mobile)
- Minimal animation on puzzle completion
- Stats page: solve count, average time by difficulty, current and best streak

## Out of scope

- Crossword format (requires clue database or editorial work — revisit as a separate idea)
- Multiplayer or social features
- Leaderboards or external accounts
- Monetization, ads, or subscriptions
- Custom puzzle construction UI
- Push notifications or reminders

## Open questions

- Should difficulty be surfaced before the user starts, or chosen upfront as the daily format?
- Is a single daily puzzle per difficulty (three tracks) better than one shared daily puzzle?
- Where does state live — localStorage only, or a lightweight backend to sync across devices?
- What does the aesthetic direction look like — minimal/typographic, retro, or something else?

## Decision log

- **Sudoku over crossword**: Research confirmed crossword requires a word/clue database or manual construction that is not feasible without significant editorial investment. Sudoku uses procedural generation libraries (e.g., sudoku-umd) and is fully automatable.
- **Personal use framing**: Research found the public casual puzzle market is winner-take-most (NYT Games at 10M+ daily players). Competing broadly is not viable. Personal tool + portfolio piece is the correct scope.
- **No backend required at launch**: Streak and progress state can live in localStorage for v1. Cross-device sync is a nice-to-have, not a requirement.
