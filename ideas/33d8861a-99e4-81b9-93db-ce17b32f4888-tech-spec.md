## Overview
Add thumbnail image support to First Read story cards. Images are extracted from RSS feed data at parse time (no scraping). Each configured feed source declares which feed field contains the image (`media:content`, `enclosure`, `itunes:image`, or `null`). The story card conditionally renders a thumbnail when an image URL is present; cards without images render identically to today — no placeholder, no layout shift. This feature is gated on a feed audit: if fewer than ~60% of configured sources expose images natively, the feature is deferred rather than built around a scraper fallback.

## Architecture
```
Feed audit (one-time) → FeedSource config updated with imageField
Feed fetch → RSS parse → extractImageUrl(item, imageField) → Story.imageUrl
Story card render → conditional thumbnail (if imageUrl present)
```
- Image URL is resolved at parse time and stored on the Story model
- No image fetching at render time beyond the URL already on the model
- The story card component gains a conditional branch for the thumbnail
- A one-time developer-facing audit script documents image field availability per source

## Stack + dependencies
- Image rendering: platform-native component (`<Image>` in React Native, `<img loading="lazy">` on web)
- No image processing library needed — thumbnails are displayed at fixed dimensions (crop, not transform)
- Caching: React Native's built-in image caching for mobile; browser cache for web
- No additional npm dependencies required

## Data model

**FeedSource config (updated)**
```ts
interface FeedSource {
  id: string
  url: string
  name: string
  schemaType: string        // from RSS normalization work
  imageField: 'media:content' | 'enclosure' | 'itunes:image' | null  // NEW
}
```

**Story (updated)**
```ts
interface Story {
  id: string
  title: string
  lede: string
  url: string
  sourceId: string
  publishedAt: Date
  imageUrl: string | null   // NEW — null if source does not provide images or field is missing
}
```

**extractImageUrl signature**
```ts
function extractImageUrl(item: RawFeedItem, imageField: FeedSource['imageField']): string | null
```

## Key implementation decisions
- **Gate on audit result:** Do not write the story card UI changes until the feed audit has run and confirmed sufficient image coverage (≥60% of sources). If coverage is insufficient, defer. This prevents building a display feature for data that doesn't exist.
- **Extract at parse time, not render time:** Avoids per-card network requests at render. The imageUrl is already on the Story when the card renders.
- **Null over placeholder:** When `imageUrl` is null, the card renders exactly as it does today — no visual gap, no broken image icon, no favicon fallback. Layout must be stable with or without the thumbnail.
- **Fixed thumbnail dimensions:** 80×80px, crop to fill (do not stretch or letterbox). Exact position (leading vs. trailing) depends on existing card layout — review before building.
- **Load failure handling:** If the image URL resolves but the request fails (404, network error, CORS), hide the image and fall back to text-only card. Log the failure; do not surface an error to the user.
- **Malformed URL handling:** If `extractImageUrl` returns a string that is not a valid URL, treat as `null` — store `null` on Story, skip rendering.

## Risks + unknowns
- **Audit may fail the 60% threshold:** If fewer than 60% of sources expose images, the feature is deferred. This is intentional and acceptable — do not proceed without audit.
- **CDN URL expiry:** Some feed image URLs (particularly from media RSS) may be short-lived signed URLs that expire before the user opens the app. Mitigation: if this is observed in audit, document affected sources — may need to re-fetch at display time for those sources specifically.
- **React Native image caching inconsistency:** Caching behavior differs between iOS and Android, and between React Native versions. Test on both platforms before shipping.
- **Layout impact:** Adding a thumbnail changes the card height or width. The "null renders as before" guarantee requires careful CSS/style scoping — ensure the conditional branch does not affect card dimensions when imageUrl is null.
- **Aspect ratio extremes:** Feed images may be very tall (portrait) or very wide (panoramic). Crop to 80×80 to contain this, but test with real feed images to confirm no visual oddities.
