## Problem
The question bank relies entirely on admin-authored questions. Users — especially domain experts on the product team — have ideas for good questions but no channel to contribute them, leaving quality content on the table and reducing community investment in the game.

## Goal
Allow any logged-in user to propose new questions for the question bank, routed through an admin review gate before going live, consistent with how existing questions are managed.

## Users
- **Primary:** Logged-in Fullstack players, especially product team members who are domain experts in the covered categories.
- **Secondary:** Admins who review, edit, approve, or reject submitted questions before they enter the live bank.

## Requirements

**Must have**
- In-app question submission form accessible from a persistent UI location (nav or profile)
- Submission fields: question text, answer, category (from existing 10), format (mc/acronym/reverse), difficulty (1–3), optional distractors for mc format
- Submitted questions enter a `Submitted` state — not live in any game mode until admin-approved
- Admin panel shows submitted questions in a dedicated review queue, separate from the existing approved question list
- Admin can approve (publish), edit then approve, or reject a submitted question

**Should have**
- Submitter receives in-app or email notification when their question is approved or rejected
- Admin can include a rejection note visible to the submitter
- Submission history page where users can see their submitted questions and current status

**Nice to have**
- Attribution: approved user-submitted questions display "Contributed by [username]" in the admin UI
- Other users can upvote pending submissions to help prioritize the admin review queue

## Out of scope
- Any path to live questions without admin review
- Dynamic AI generation of questions
- Bulk submission via CSV or external tooling
- Submitters editing a question once it has been reviewed

## Open questions
- Where does the submission entry point live — after answering a question, in the main nav, or on the profile page?
- Does the existing `/admin/questions` route expand to handle submissions, or does this need a new route (e.g., `/admin/submissions`)?
- Should submitters be able to edit or retract a submission before it enters the admin review queue?

## Decision log
- Admin review gate required before questions go live — mirrors how all 213 existing questions are authored and approved
- No direct-to-live path even for trusted users in v1 — simpler to relax the gate later than to roll it back
- In-app form preferred over an external link (e.g., Google Form) to keep the experience cohesive and enable status tracking
