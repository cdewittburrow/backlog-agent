## Problem
Players encounter questions that feel wrong, poorly worded, or AI-generated slop with no way to signal quality issues. This creates silent degradation of the question bank and frustrates users who notice problems but have no recourse.

## Goal
Add a lightweight, in-session feedback mechanism that lets users flag question quality issues, surfacing actionable signal to admins without disrupting gameplay flow.

## Users
- **Primary:** All logged-in Fullstack players — especially core users like Chase's friend group who play regularly and have strong opinions about question quality.
- **Secondary:** Admins managing the question bank at `/admin/questions`.

## Requirements

**Must have**
- A per-question feedback button (thumbs down or flag icon) accessible during Daily Five, Freeplay, and Level Set
- Signal tied to `(user_id, question_id)` — one submission per user per question to prevent spam
- Admin panel shows aggregate flag count per question
- Flagged questions (above a configurable threshold) are visually distinguished in the admin queue

**Should have**
- Optional free-text reason when flagging (e.g., "wrong answer", "confusing wording", "AI slop")
- Admin actions: dismiss flags (mark reviewed) or unpublish the question
- Flag count displayed as an aggregate in admin UI — not attributable to individual users

**Nice to have**
- Auto-hide question from new sessions after N flags (configurable threshold, default 5)
- Thumbs up (positive signal) in addition to thumbs down to identify high-quality questions

## Out of scope
- Public display of feedback counts to non-admin users
- Commenting or discussion threads on questions
- Feedback affecting XP weighting or difficulty rating
- AI-powered automatic quality scoring

## Open questions
- Should the feedback button appear while the user is still deciding, or only after they've answered?
- What is the initial flag threshold for auto-hide, or should auto-hide be omitted from v1?
- Does a flagged question need to be manually unpublished, or can it be auto-hidden pending review?

## Decision log
- Aggregate signal (not per-user display) to preserve user privacy in admin view
- One signal per `(user, question)` pair to prevent vote-stuffing
- Free-text reason optional rather than required — friction reduces signal volume
- Fits within existing admin UI at `/admin/questions` rather than requiring a new route
