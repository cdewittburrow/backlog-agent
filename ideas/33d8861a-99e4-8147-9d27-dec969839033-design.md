## Overview
We are normalizing how RSS feed content is extracted and displayed in First Read. The problem is not a UI design problem — it is a data pipeline design problem. The output of this work is a consistent `lede` field that the existing UI can render without changes.

## User flows

**Reading the feed (after normalization):**
1. User opens First Read
2. Feed loads — each story shows headline + lede
3. Every lede is ~2–3 sentences regardless of source
4. User scans and taps in or dismisses

No new UI interactions — this is invisible to the user if done correctly.

## Information architecture

**Feed ingestion layer (new/changed):**
- `FeedSource` config: includes `schema_type` field (e.g., `standard`, `axios_full_body`, `media_rss`)
- `normalizeContent(item, schemaType)`: extracts and truncates to lede
- Output: a single `lede` string, HTML-stripped, ≤300 chars, ending at a sentence boundary

**Story data model (unchanged surface, updated content):**
- `story.lede` — now guaranteed to be normalized
- Raw feed data can optionally be retained for debugging

## Interaction patterns

No new interactions. The normalization happens at fetch/parse time before stories are stored or displayed. From the user perspective, the feed simply works correctly.

**Edge cases surfaced during normalization:**
- Empty lede after stripping → fall back to headline
- Lede identical to headline → display lede only once (suppress redundant subtitle)

## Edge cases + empty states

- **Source has no content field at all:** use `title` only, no lede shown
- **Content is very short (<50 chars):** display as-is, no truncation
- **Content is HTML with inline styles/scripts:** strip all tags, decode entities
- **New source added without schema_type set:** default to `standard` behavior, flag in logs

## Open questions

- Should schema_type be hardcoded per source or auto-detected on first fetch?
- Does First Read have a concept of "feed source configuration" already, or does this need to be built?
- Is there a test fixture of real feed payloads to validate normalization against?
